# Contributing

Thank you for your interest in contributing to **GitHub Actions Fundamentals**.

This repository is an educational laboratory designed to help learners understand the fundamentals of GitHub Actions and Continuous Integration.

Contributions that improve the quality, accuracy, and educational value of this project are welcome.

---

# Ways to Contribute

You can contribute by:

- Fixing documentation issues
- Improving workflow examples
- Correcting typos or grammar
- Enhancing diagrams
- Adding beginner-friendly explanations
- Improving repository organization
- Reporting bugs or inconsistencies

---

# Getting Started

1. Fork the repository.
2. Create a new feature branch.

```bash
git checkout -b feature/my-improvement
```

3. Make your changes.
4. Commit using a descriptive message.

Example:

```bash
git commit -m "Improve workflow documentation"
```

5. Push your branch.

```bash
git push origin feature/my-improvement
```

6. Open a Pull Request.

---

# Contribution Guidelines

Please ensure that your contribution:

- Maintains a clear educational focus.
- Uses simple and readable language.
- Follows existing formatting and structure.
- Includes documentation updates when applicable.
- Does not introduce unnecessary complexity.

---

# Workflow Changes

If you modify a GitHub Actions workflow:

- Verify that the workflow executes successfully.
- Use official GitHub Actions whenever possible.
- Keep workflows easy to understand.
- Avoid unnecessary dependencies.

---

# Documentation Standards

Documentation should:

- Use Markdown.
- Include meaningful headings.
- Provide practical examples.
- Explain technical concepts clearly.

---

# Reporting Issues

If you discover an error or have a suggestion, please open a GitHub Issue describing:

- The problem
- Steps to reproduce (if applicable)
- Expected behavior
- Proposed improvement

---

# Code of Conduct

Please be respectful and constructive when interacting with other contributors.

By participating in this project, you agree to follow the project's Code of Conduct.

---

# Thank You

Thank you for helping improve this educational project and supporting the open-source community.
