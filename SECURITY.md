# Security Policy

## Overview

Thank you for helping improve the security of **GitHub Actions Fundamentals**.

This repository is an educational project designed to teach the fundamentals of GitHub Actions and Continuous Integration. Although it is not intended for production use, responsible disclosure of security issues is appreciated.

---

# Supported Versions

Only the latest version available on the `main` branch is actively maintained.

| Version | Supported |
|---------|-----------|
| main | ✅ Yes |
| Older commits | ❌ No |

---

# Reporting a Security Vulnerability

If you discover a security issue related to this repository, please report it responsibly.

Please include as much information as possible:

- Description of the issue
- Steps to reproduce
- Potential impact
- Suggested remediation (if available)

Avoid publicly disclosing vulnerabilities before they have been reviewed.

---

# Scope

This policy applies to:

- GitHub Actions workflows
- Documentation
- Repository configuration
- Scripts included in this project

The following are generally considered out of scope:

- Vulnerabilities affecting GitHub itself
- Vulnerabilities in third-party services
- Security issues unrelated to this repository

---

# Security Best Practices

This repository follows several security best practices, including:

- No hardcoded credentials
- No secrets committed to the repository
- Use of official GitHub Actions whenever possible
- Version-pinned Actions
- Clear documentation of workflow behavior
- Publicly reviewable source code

---

# Educational Notice

This project is intended for learning purposes.

The examples included in this repository demonstrate GitHub Actions concepts and should be reviewed and adapted before being used in production environments.

---

# Responsible Disclosure

Please report vulnerabilities responsibly and allow reasonable time for investigation before making them public.

Constructive feedback and security improvements are always welcome.

# Acknowledgements

Thank you for contributing to a safer and more secure open-source community.
